# SpecKit to Ralph++ PRD Converter

Convert `.speckit/features/{feature}/tasks.md` into a `prd.json` that `/ralph-loop` can execute.

## Dynamic Context

- Available features: !`ls .speckit/features/ 2>/dev/null || echo "NO_FEATURES"`
- Current directory: !`pwd`

## Step 1: Find and Read Tasks

If a feature name was provided as argument, read its tasks:
- `.speckit/features/{feature}/tasks.md`
- `.speckit/features/{feature}/plan.md`
- `.speckit/features/{feature}/spec.md`

If no argument, list available features from the context above and ask the user.

## Step 2: Parse Tasks

The tasks.md format uses **Phases and Tasks**:

```markdown
## Phase 1: Phase Name (Tasks X-Y)

### Task 1: Task title
**File:** `path/to/file.ts`
**Action:** Create | Modify
**Details:**
- Detail 1
- Detail 2
```

Map each task:

| tasks.md Field | prd.json Field |
|----------------|----------------|
| Task N: Title | id: N, title |
| File: path | filesAffected |
| Action: Create/Modify | description context |
| Details (bullets) | description + acceptanceCriteria |
| Phase grouping | dependency inference |

## Step 3: Extract Dependencies

From the `## Dependencies` section at bottom of tasks.md:
```
Task 1 → Task 3 (reason)
```
Maps to: story 3 has `dependencies: [1]`

Tasks in later phases implicitly depend on earlier phases.

## Step 4: Detect Parallel Groups

From the `Parallelizable:` section:
```
Parallelizable: Tasks 1+2 (no deps)
```
Mark with `"parallel": true` and same `"group"` number.

## Step 5: Generate PRD JSON

Write `prd.json` with this structure:

```json
{
  "projectName": "feature-name",
  "config": {
    "qualityGates": {
      "enabled": true,
      "typecheck": "npx tsc --noEmit",
      "lint": "npx next lint",
      "test": "npx vitest run",
      "command": "npx tsc --noEmit && npx next lint && npx vitest run && npm run quality:jscpd && npm run quality:knip"
    },
    "projectRoot": "/absolute/path/to/project",
    "constitutionPath": ".speckit/constitution.md",
    "specPath": ".speckit/features/{feature}/spec.md",
    "planPath": ".speckit/features/{feature}/plan.md"
  },
  "stories": [
    {
      "id": 1, "title": "...", "description": "...",
      "acceptanceCriteria": ["..."],
      "filesAffected": ["..."],
      "dependencies": [], "parallel": false, "group": null,
      "priority": 1, "complexity": "low",
      "passes": false, "notes": ""
    }
  ],
  "metadata": {
    "createdAt": "ISO timestamp", "source": "speckit",
    "totalTasks": 0, "phases": 0,
    "tasksPath": ".speckit/features/{feature}/tasks.md"
  }
}
```

### Acceptance Criteria Generation

For each task, generate from:
1. **File existence**: "File {path} exists" (for Create actions)
2. **Functions/components**: Extract names from Details
3. **Behavior**: What the code should do
4. **Integration**: "Existing functionality preserved" (for Modify actions)
5. **Spec-driven tests**: If `tests/speckit/{feature}/` exists, add criterion: "All tests in tests/speckit/{feature}/ pass"

### Spec-Driven Test Linking

If the tasks.md references spec-driven test files (generated by `/speckit-tasks`):
- Add `"speckitTests"` field to each story pointing to its test file(s)
- Add to story description: "IMPORTANT: You MUST make the tests in {test-file} pass. Do NOT modify files in tests/speckit/."
- Example:
```json
{
  "id": 3,
  "title": "Create Post action",
  "speckitTests": ["tests/speckit/posts-crud/us-001-create.test.ts"],
  "description": "... IMPORTANT: You MUST make the tests in tests/speckit/posts-crud/us-001-create.test.ts pass. Do NOT modify files in tests/speckit/."
}
```

## Step 6: Create Session Directory

```bash
SESSION_ID=$(date +%s)000
mkdir -p .ralph++/sessions/$SESSION_ID
```

Write prd.json and create progress.txt:
```
# Progress Log - {feature-name}

Session: {SESSION_ID}
Feature: {feature-name}
Created: {ISO timestamp}
```

## Step 7: Report

```
PRD created: .ralph++/sessions/{session-id}/prd.json

Feature: {name}
Stories: {total} (from {phases} phases)
Parallel groups: {count}
Sequential: {count}

Ready! Run: /ralph-loop {session-id}
```

## Validation

Before writing, verify:
- All tasks included, IDs match task numbers
- Dependencies extracted correctly
- Parallel groups identified
- All `passes` fields are `false`
- File paths are exact
- acceptanceCriteria are concrete and verifiable

**NOW**: Read the tasks file, parse all tasks, create prd.json and session directory.
