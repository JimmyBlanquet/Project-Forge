# /pf-cross-review-iterate — Iterative auto-stabilization loop (V4)

## Mission

Run `/pf-cross-review` (V3.3, cross-school N0+N1+N2) **multiple times in sequence** until the set of findings **stabilizes** — i.e. no new finding for K consecutive rounds. Detects when iteration is making things worse (`diverging`) and stops with alarm. Implements safety monotonicity from SCAFFOLD-CEGIS (arXiv 2603.08520) : *"43.7% of GPT-4o chains degrade after 10 rounds without hard constraints, mitigeable to 2.1% via gating"*.

## When to use

- Tu viens d'appliquer plusieurs fixes successifs (V2→V3→V3.1→V3.2 pattern) et tu veux vérifier que la suite stabilise vraiment
- Tu refactor un module sensible et veux confirmer qu'aucun nouveau bug n'est introduit
- Tu veux capitaliser un audit cross-school (1 review) en série stabilisée (N reviews)

## When NOT to use

- Single small commit (overhead inutile, 1 round suffit)
- Range > 50 commits (trop large pour boucle, découper)
- Diff > ~100 KB chars (~25K tokens) : N2 saturera son ctx-size à chaque round ; utiliser `--no-n2` ou découper en sous-ranges
- Codex quota épuisé (boucle multipliera la charge)
- Budget < $5 (au moins 2 rounds × N0+N1+N2 = ~$2-4 selon volume)

## Arguments

```
/speckit.pf-cross-review-iterate.iterate [<git-range>] [--max-rounds N] [--stagnation-k N] [--cost-cap-usd N] [--no-n2] [--n0-only]
```

- `<git-range>` (optional) — same auto-detection as `/pf-cross-review`
- `--max-rounds N` (default 5) — hard cap iterations
- `--stagnation-k N` (default 2) — K rounds consécutifs sans new finding = STOP `stable`
- `--cost-cap-usd N` (default 5.00) — hard cap budget cumulé
- `--no-n2` / `--n0-only` — passés à `/pf-cross-review` (cf doc V3.3)

## Workflow

### Setup (Round 0)

```bash
SESSION_DIR=".cross-review-iterate/$(date +%s)"
mkdir -p "$SESSION_DIR"

# Configuration (with defaults)
MAX_ROUNDS="${MAX_ROUNDS:-5}"
STAGNATION_K="${STAGNATION_K:-2}"
COST_CAP_USD="${COST_CAP_USD:-5.00}"

# State trackers
STAGNANT_COUNT=0   # rounds consécutifs sans new finding
COST_USED=0.00
EXIT_REASON=""
```

Tell user: **"Starting iterative cross-review on $RANGE — max $MAX_ROUNDS rounds, stop after $STAGNATION_K consecutive stable rounds, budget cap \$$COST_CAP_USD."**

### Round Loop

For `round in 1..MAX_ROUNDS`:

#### 1. Cost gate (preventive)

If `COST_USED >= COST_CAP_USD` → set `EXIT_REASON="budget"` and break.

#### 2. Run /pf-cross-review V3.3

Invoke the underlying skill with the same range. Capture the output report path (e.g. `docs/plans/YYYY-MM-DD-cross-review-<slug>.md`).

```
Use Skill tool with: speckit.pf-cross-review.cross-review
Pass: <RANGE> [--n0-only|--no-n2 si flags]
```

Wait for completion. Note the elapsed time.

#### 3. Parse findings → fingerprints

Read the consolidation doc generated by /pf-cross-review. Extract every finding from the consolidation table (skip the `Faux positifs rejetés` appendix). For each finding :

- Extract `file:line` (from "File:" or table column)
- Extract `claim` (from "Issue" / "claim" / first sentence of finding)

Compute fingerprint via the helper :

```bash
for each finding (file_line, claim):
  fp=$(scripts/cross-review-iterate/fingerprint.sh "$file_line" "$claim")
  echo "$fp" >> "$SESSION_DIR/round-$round.fingerprints"
  echo "$file_line|$claim" >> "$SESSION_DIR/round-$round.findings.txt"
done
```

Sort + uniq both files at the end :
```bash
sort -u -o "$SESSION_DIR/round-$round.fingerprints"{,}
sort -u -o "$SESSION_DIR/round-$round.findings.txt"{,}
```

#### 4. Cost tracking

Estimate cost of this round :
- N0 (Anthropic Opus sub-agent) : ~$0.50-1.00 typical (varies with code size)
- N1 (codex/OpenAI) : ~$0 marginal (sur ChatGPT plan)
- N2 (local LLM (your llama-swap / vLLM / Ollama instance)) : $0 marginal (GPU déjà payé)

Accumulate `COST_USED` (rough estimate, log-only). Real billing is plan-side.

#### 5. Diff vs previous round (skip if round 1)

If `round >= 2` :

**V4.1 (recommandé) — semantic dedup via LLM-as-judge** : utiliser `diff-rounds-semantic.sh` qui combine fingerprint exact (fast path) + LLM-as-judge (devstral-small-2 par défaut) sur les findings sans match exact. Détecte la **convergence sémantique** (même bug, wording différent) qu'un fingerprint sha256 strict rate.

```bash
# V4.1 sémantique — cible : findings au format "file:line|claim" (pas fingerprint)
DIFF_JSON=$(scripts/cross-review-iterate/diff-rounds-semantic.sh \
    "$SESSION_DIR/round-$((round-1)).findings.txt" \
    "$SESSION_DIR/round-$round.findings.txt")
VERDICT=$(echo "$DIFF_JSON" | jq -r '.verdict')
NEW_COUNT=$(echo "$DIFF_JSON" | jq -r '.new')
DROPPED_COUNT=$(echo "$DIFF_JSON" | jq -r '.dropped')
EXACT_STABLE=$(echo "$DIFF_JSON" | jq -r '.exact_stable')
SEMANTIC_STABLE=$(echo "$DIFF_JSON" | jq -r '.semantic_stable')
```

**Fallback V4 basique** (si local LLM endpoint N2 indisponible) — `diff-rounds.sh` sur fichiers fingerprints exacts seulement. Risque : 2 wordings du même bug = 2 fingerprints distincts → faux verdict "converging" alors que sémantiquement stable. À utiliser uniquement en mode dégradé.

```bash
# V4 basique fallback (mode dégradé sans LLM-as-judge)
DIFF_JSON=$(scripts/cross-review-iterate/diff-rounds.sh \
    "$SESSION_DIR/round-$((round-1)).fingerprints" \
    "$SESSION_DIR/round-$round.fingerprints")
```

**Cas d'usage observé (dogfood V4 sur lui-même 2026-05-03)** : devstral-small-2 R1 vs R2 sur même cluster = 5 findings identiques sémantiquement avec wording différent. V4 basique disait `new=5/dropped=5/stable=0/converging` (faux). V4.1 sémantique dit `new=0/dropped=0/stable=5/STABLE` (correct, exact_stable=0 + semantic_stable=5 via LLM-as-judge). C'est exactement la valeur de V4.1.

Print to user: `Round $round: $VERDICT — $NEW_COUNT new, $DROPPED_COUNT dropped, $STABLE_COUNT stable`.

Append to `$SESSION_DIR/log.jsonl` :
```json
{"round":N, "verdict":"...", "new":N, "dropped":N, "stable":N, "elapsed_s":N, "cost_estimate_usd":N}
```

#### 6. Apply convergence rules

```
case "$VERDICT" in
  stable)
    STAGNANT_COUNT=$((STAGNANT_COUNT + 1))
    if [ "$STAGNANT_COUNT" -ge "$STAGNATION_K" ]; then
      EXIT_REASON="converged"; break
    fi
    ;;
  converging)
    STAGNANT_COUNT=0  # reset (qualité monte, pas encore stable)
    ;;
  growing)
    STAGNANT_COUNT=0
    ;;
  diverging)
    EXIT_REASON="diverging"; break  # alarme spec drift
    ;;
esac
```

### Final summary

```
✅ /pf-cross-review-iterate done.
   Exit reason : $EXIT_REASON
   Rounds run  : $ROUND
   Cost (est)  : ~$$COST_USED
   Session dir : $SESSION_DIR/
   Final report: <last cross-review doc path>

   Per-round summary :
   $(cat "$SESSION_DIR/log.jsonl" | jq -c '{round, verdict, new, dropped}')
```

Then summarize the **last round's findings** (the converged or diverging set) for the user to act on.

## Exit reason interpretation

| Exit reason | Meaning | What to do next |
|---|---|---|
| `converged` | $STAGNATION_K rounds consécutifs sans new finding → set stable | Apply final round findings as plan, ship. Real signal of stability. |
| `diverging` | New findings > 1.5× previous → spec drift suspecté | **STOP, escalate human**. Inspect log : retours en arrière possible via `git tag cross-review-r{N}` ? |
| `budget` | Cost cap hit avant convergence | Soit augmenter `--cost-cap-usd`, soit accepter le snapshot du dernier round comme "best effort" |
| `max_iter` | $MAX_ROUNDS hit sans converger | Inspect : peut-être le code a vraiment beaucoup de bugs. Considérer fix par batches au lieu de re-reviewer. |

## Anti-patterns à éviter

- **Ne PAS lancer sans avoir d'abord testé `/pf-cross-review` simple sur le range** — vérifier que les pré-requis (codex, local LLM, sub-agent) marchent en 1 round avant d'investir N rounds
- **Ne PAS utiliser `--max-rounds > 10`** — au-delà, retours décroissants prouvés (Self-Refine NeurIPS 2023, SCAFFOLD-CEGIS)
- **Ne PAS interpréter `converged` comme certitude absolue** — c'est "stable sur K rounds avec ce reviewer set". Un cycle 6 ou un autre reviewer set pourrait trouver autre chose
- **Ne PAS auto-fixer entre rounds** — V4 ne fait QUE détecter la convergence, pas fixer les bugs. Les fixes restent décision humaine entre 2 invocations

## Session directory layout

```
.cross-review-iterate/<timestamp>/
├── round-1.fingerprints       # 1 fp sha256 par ligne, sort-u (V4 fallback)
├── round-1.findings.txt       # 1 "file:line|claim" par ligne, sort-u (V4.1 semantic)
├── round-1.report.md          # copie du doc /pf-cross-review (référence)
├── round-2.fingerprints
├── round-2.findings.txt
├── round-2.report.md
├── ...
└── log.jsonl                  # log structuré : 1 ligne JSON par round
```

Cette session_dir est gitignorée (sauf décision explicite de commit pour traçabilité audit).

## Cas limites

| Cas | Comportement |
|---|---|
| /pf-cross-review fail dans round N | Capture l'erreur, log avec `verdict:"error"`, STOP avec EXIT_REASON="error" |
| Tous les modèles N1+N2 unavailable | Mode dégradé N0-only ; si même N0 fail → EXIT_REASON="error" |
| Round 1 produit 0 finding | Conv déjà au round 1 ; STOP avec EXIT_REASON="empty" (cas limite : code propre OU reviewer mal configuré) |
| jq ou helper script absent | EXIT_REASON="missing_tool" + escalade |
| Range invalide | Erreur avant la boucle (cf /pf-cross-review V3.3 Step 1) |

## Output expected

- 1 fichier `.cross-review-iterate/<ts>/log.jsonl` (per-round structured log)
- 1 fichier `.cross-review-iterate/<ts>/round-N.report.md` par round (copie consolidation)
- Console summary final avec EXIT_REASON + per-round table
- 0 modification de code (c'est un READ-only orchestrator)
- 0 commit auto

## Pré-requis

- `/pf-cross-review` V3.3+ installé (ou compatible — fournit consolidation doc)
- `scripts/cross-review-iterate/fingerprint.sh` exécutable
- `scripts/cross-review-iterate/diff-rounds.sh` exécutable
- `jq` installé pour parsing JSON output diff-rounds
- Budget plan Anthropic + ChatGPT (pour N0+N1) ou flag --no-n2 / --n0-only si tout OFF
