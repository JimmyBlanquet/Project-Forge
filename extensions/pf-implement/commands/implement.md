# Project-Forge Implementation (Ralph++)

Launch autonomous implementation using Ralph++ from a prd.json file.

## Prerequisites

- `prd.json` — generated by `/speckit.pf.convert` from tasks.md
- `progress.txt` — in the Ralph++ session directory
- `claude` CLI — available in PATH

## Usage

```
/speckit.pf.implement [prd-path]
```

If no path provided, auto-detect from `.ralph++/sessions/*/prd.json` (most recent).

## Step 0: Locate PRD

1. If path argument provided, use it
2. Otherwise, find the most recent `prd.json` in `.ralph++/sessions/`
3. Validate the PRD is valid JSON with `config` and `stories` fields
4. Read `progress.txt` to determine current state

## Step 1: Analyze Dependencies

Read the PRD and identify:
- **Independent stories** (no `depends_on`) — can run in parallel
- **Dependent stories** — must wait for prerequisites
- **Phases** — group stories into execution batches

## Step 2: Execute Implementation

For each batch of independent stories:

1. Create a feature branch: `feat/{story-slug}`
2. Launch implementation via `claude -p` with the story context
3. Run quality gates after each batch (not per story):
   - `pnpm typecheck`
   - `pnpm lint`
   - `pnpm test:run`
   - `pnpm quality:jscpd`
   - `pnpm quality:knip`
4. Update `progress.txt` with completed stories
5. Commit and push the feature branch
6. Create a PR via `gh pr create`

## Step 3: Report

```
Implementation Complete

  Stories: {completed}/{total}
  Branches: feat/{story-1}, feat/{story-2}, ...
  PRs: #123, #124, ...
  Quality Gate: PASS

Next steps:
  - Review PRs
  - Merge when CI passes
```

## Options

- `--auto-merge` — Enable `gh pr merge --auto --squash --delete-branch` for hands-free
- `--budget <amount>` — Max spend per story (default: $2)
- `--model <model>` — Claude model to use (default: sonnet)
- `--parallel <n>` — Max parallel agents (default: 3)

## Error Handling

If a story fails:
- Log the error in `progress.txt`
- Skip dependent stories
- Continue with independent stories
- Report failures in the final summary
